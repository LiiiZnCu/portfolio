# 作品集首页编辑式展厅重排

## 目标

把现有首页从“信息面板”改成更接近成熟设计师官网的编辑式项目展厅。页面首先建立个人身份，其次让项目图片成为主角，再按真实设计过程解释项目。

## 视觉方向

- 纯白背景、黑灰文字，东方暗红只用于当前项目、编号和少量过程线索。
- 保留工业策展感，但减少连续边框、标签和表格感。
- 使用不对称网格、大图、窄文字栏和明显留白。
- 只使用用户提供的彩色素材，不生成图片。
- `PORTFOLIO` 只作为小型站点名称，不再作为占据页面的大标题。

## 首页顺序

### 1. 个人简介

- 首屏显示姓名、英文名、身份“智能交互设计学生”、一句简短介绍和彩色个人照片。
- 院校、方向、所在地和邮箱作为一行简洁资料。
- 工具、经历、语言等次要信息收进可展开的个人档案，避免首屏像简历表格。

### 2. 项目目录

- 首屏下方直接出现 8 个项目目录。
- 桌面端左侧为固定项目列表，中间为当前项目主图，右侧为项目名称、类别和一句话简介。
- 手机端以“当前项目 + 展开项目目录”代替横向挤压的 8 个标签。
- 支持点击、键盘方向键和“上一个 / 下一个项目”切换。
- 当前项目使用暗红编号和状态线标记，不使用大面积色块。

### 3. 当前项目内容

阅读顺序固定为：

1. 项目主图
2. 项目概述
3. 问题与目标
4. 研究或设计过程
5. 方案与关键设计
6. 最终结果与其他媒体

项目切换后只更新当前项目内容，不跳到新的长页面。

## 项目素材组织

### 素材选择原则

- 主图优先使用完整成果图。
- 从 PPT、PDF 中提取用户旅程、用户画像、信息架构、任务流程、草图、结构爆炸图和测试过程。
- 不直接把整张汇报展板当作普通图库图片；需要时裁出其中真正承担解释作用的内容。
- 重复角度、重复合成图和低信息量图片删除。
- 每个项目控制在 4—7 个有效内容节点，而不是堆放全部素材。
- 原始数据、研究结论和项目成果不编造；缺少说明时只补充叙事连接句。

### 各项目重点

- SportLoop：问题发现、三类用户、信息架构、核心任务流程、学生端与管理端。
- VERSA：用户痛点、用户旅程、结构爆炸、双加热模式、CMF、使用视频。
- 弧刃：造型推演、建模、材质灯光、最终场景。
- 戏曲塔罗：文化映射、人物参考、手绘重构、卡面系统、包装、过程视频。
- 阿根廷信息可视化：叙事逻辑、信息编码、局部数据关系和最终双展板；展板只作为结果收尾。
- 机械项目：轨道机制、草图、制作测试、三叶轮与齿轮结构、攀爬结果。
- 生长中的记忆：材料选择、空间关系、日间和亮灯状态。
- AIGC 海报：双视觉路线、生成与筛选方法、两张最终海报。

## 交互

- 项目切换采用 180—240ms 淡入和轻微位移，避免突然替换。
- 主图可通过缩略图或前后按钮切换。
- 过程内容使用纵向章节，不做弹窗。
- 视频保留原生控制，并显示明确的视频标签。
- URL 保留项目锚点，刷新后继续显示当前项目。
- 所有点击区域至少 44px，保留键盘焦点和 reduced-motion 支持。

## 响应式

- 1440px：三栏展厅，图片占主要宽度。
- 768—1180px：项目目录横向或顶部排列，图片与说明上下分层。
- 375—767px：项目菜单折叠，主图优先，过程说明单列阅读。
- 不允许页面产生横向溢出。

## 验证

- 8 个项目均可选择，并保持深链接。
- 每个项目至少包含成果图和一个解释设计过程的内容节点。
- VERSA 与戏曲塔罗视频可加载播放。
- 不存在重复素材、生成图片或误归类图片。
- 检查 375px、768px、1440px 三种宽度。
- 自动测试、生产构建、浏览器错误和资源加载全部通过。
