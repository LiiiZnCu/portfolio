# 作品集媒体完整性与一屏浏览设计

## 目标

- 桌面端和手机端的项目主媒体均能在当前视口内完整看完，不裁切。
- SportLoop 的手机界面截图不再纵向占据多屏，保持完整比例并居中展示。
- 接入 SportLoop PPT 中真实存在的关键交互动效，不用静态图冒充交互。
- 重新核对 8 个项目，只补充有解释价值、来源真实且不重复的素材。

## 媒体规则

- 项目主媒体最大高度：桌面端 `calc(100dvh - 12rem)`，手机端 `62dvh`。
- 横图、竖图、视频都使用 `object-fit: contain`，容器跟随可用视口，不裁切。
- 背景保持透明，不增加图片外框。
- 过程图保持自然比例，但手机端最大高度不超过 `68dvh`。

## SportLoop

- 静态成果保留：首页、器材列表、器材详情、管理员端。
- 从 PPT 提取并压缩以下交互：
  1. 点击与按钮反馈。
  2. 管理端环形滑动。
  3. 管理端下拉刷新。
  4. 器材筛选。
  5. 列表到详情再到借用的流程。
  6. 我的借用与状态反馈。
- 相同页面的多个 GIF 合并为一个交互组，避免重复。
- 补充真实测试反馈与迭代结论，不编造测试数据。

## 其他项目

- VERSA：保留用户旅程、结构、旋钮交互、爆炸图、材质和演示视频。
- 弧刃：只使用两个不同场景视角，不补不存在的建模过程。
- 戏曲塔罗：保留原型参考、手绘重构、边框系统、完整卡组、包装和制作视频。
- 阿根廷信息可视化：保留系统路径、信息编码、结构关系和数据对照，不堆完整展板。
- 机械项目：保留草图、制作测试、轨道机构、机器人整机、齿轮和攀爬结果。
- 生长中的记忆：保留日间、材料细节和亮灯状态。
- AIGC 海报：只展示两条最终视觉路线，不虚构提示词和筛选过程。

## 验证

- 1440px、768px、390px 三种宽度逐项检查 8 个项目。
- 所有主媒体均完整落在视口高度内。
- 所有图片与视频正常加载，保持原始比例。
- 同一项目不引用内容相同的重复素材。
- SportLoop 六组动效均可播放。
