# 项目协作规则

- 用简洁中文回复，先说结论。
- 开始任务时只读 `CONTEXT.md` 和与任务直接相关的文件。
- 未经明确同意，不创建分支或 worktree，不删除文件，不操作数据库。
- 简单修改直接执行；只有重大重构或多个相互独立的任务才制定计划、使用子代理。
- 从根因解决问题，不添加降级、兜底或临时补丁。
- Python 优先使用项目内 `.venv`；保持文件原有 UTF-8 编码。
- 修改后检查差异，运行相关测试和构建，只报告通过或失败数量。
- 每次阶段完成后精简更新 `CONTEXT.md`；历史内容通过 Git 记录查阅。
- 只有功能模块、依赖、运行方式或重要设计决定变化时，才更新 `README.md` 或 `ARCHITECTURE.md`。
- 未经用户要求，不主动扩展任务范围。
