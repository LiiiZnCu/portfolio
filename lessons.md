# Lessons

- 用户确定的“工业策展感 + 东方暗红/像素点缀”不能被解释为黑底暗黑科技风。
- 未经用户明确同意，不生成或补充项目图片；只使用用户提供的真实素材。
- 素材不足时采用无图排版，直接暴露缺口，不使用占位图或推测性封面。
- 主视觉以灰白黑为底；暗红是主要点缀，金色与少量蓝色只用于材料标签和过程线索。
- 页面应像“产品设计展 + 个人实验档案”，以大图、编号、网格、材料标签和过程线索组织内容，避免普通简历页。
- 用户要的是明确的纯白底，不是偏灰、偏米或大面积暗色。
- 首页唯一主标题使用 `PORTFOLIO`，不再加入抽象口号抢占首屏。
- 首屏必须直接说明姓名、院校、专业方向、能力与所在地，不能只留一句泛化介绍。
- 项目清单必须完整展示 8 项，包括独立的“弧刃”三维练习与 AIGC 海报实验。
- 用户已经提供了完整的“作品 2”整合目录；不能只检查当前代码仓库就判断项目没有真实图片。
- 开始作品集首页工作前，应读取同项目中“整理作品集素材”和“整理作品集项目清单”线程，复用已确认的分类与命名。
- 完整项目清单是 8 项；“弧刃”小刀建模为独立项目，两组空气炸锅渲染都归入 VERSA。
- “雕塑小模型制作”目录混有《生长中的记忆》和《折羽之间》两件不同作品，不能把 HEIC 雕塑图误归为记忆装置。
- 个人档案必须先于 `PORTFOLIO` 标题出现，首屏先回答“我是谁”，再进入作品索引。
- 项目首页使用可选择的展厅结构，一次聚焦一个项目，不能让用户连续下滑八张长卡片。
- 项目素材应尽量完整接入原图和原视频；空气炸锅与戏曲塔罗都要展示真实过程视频。
- 人像与作品图保持原始彩色，不擅自统一成黑白或降低色彩。
- 项目介绍优先从 PPT、PDF 和项目文件提取；素材说明不足时只补充不含虚假数据的叙事文案。
- 首屏身份标题必须短而直接，具体能力放在下方说明和档案字段中，不在大标题重复罗列。
- 项目说明不能只靠概述文字和成果图库；应从 PPT、PDF 中提取用户旅程、信息架构、草图、结构和测试过程来说明设计逻辑。
- 完整展板只适合作为项目结果收尾，不能替代过程内容；重复角度和重复合成图应删除。
- GitHub Pages 项目站部署在 `/portfolio/` 子目录，动态生成的图片与视频地址必须使用 Vite `BASE_URL`，不能直接使用 `/media/...` 根路径。
- 项目图片不能套统一宽高比；应让容器跟随原图高度，避免 `contain` 留边、`cover` 裁切和移动端父容器高度错误。
- 项目主媒体连同图片说明和切换按钮，必须在常见电脑与手机视口的一屏内完整显示，不能只限制图片本身。
- PPT 中存在多组交互动效时，应按实际交互任务完整筛选接入，不能只挑少数几张代表性动图。
- 复核项目素材时要逐项说明取舍：有真实视频和过程图就使用；只有重复图、整板或无法确认归属的素材就不展示。
- 个人核心信息不使用折叠交互，能力、工具、经历和语言应直接展示，减少一次无意义点击。
- 项目索引中的编号、标题和分类必须保留统一内边距，选中状态不能通过改变左边距造成文字贴边或跳动。
- 手机界面成果图需要统一比例和外观；若只有原始媒体且没有可运行网页，再使用不裁内容的画布规范方式。
- 用户提供可运行网页时，手机展示图应从网页按目标视口重新截图；不能拿旧比例素材直接套设备壳。设备边框只负责圆角和细线，不应再添加厚黑内垫或重复的灵动岛。
- 设备外框规范必须同时覆盖静态截图和全部动图；边框粗细与灵动岛应作为统一容器样式逐项验证，不能只检查首页主图。
- 非对称双列布局交换内容顺序时必须同步交换列宽；只改 `order` 会让偶数项图片落入文字窄列。
- 过程块有真实测试数据时不能只放结论文字；应把 PPT 中的原始数值做成可读图表填入宽图列，避免无意义留白。
- 可运行的原网页存在时，交互动效必须按目标手机视口重新操作并逐帧录制；不能缩放、拉伸或补边旧动图。桌面主图尺寸不能用视口高度减固定大值，否则矮屏会把手机图压得过小。
