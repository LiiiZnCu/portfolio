# Lessons

- 网站保持纯白底、黑灰文字和少量东方暗红点缀，不改成暗黑科技风。
- 只使用用户提供或项目中已有的真实素材，不生成、不猜测、不使用占位图。
- 首页先展示个人身份和档案，再进入八个项目的可选择展厅。
- 项目内容按“问题 → 研究/过程 → 方案 → 结果”组织，避免堆放完整展板和重复图片。
- 图片按原始比例展示；手机界面素材必须使用真实目标视口截图，不能拉伸旧图。
- GitHub Pages 部署在子目录，媒体路径必须通过站点基础路径生成。
- SportLoop 素材必须核对最新网页，并验证图片加载、动画帧时长和手机安全区。
- 项目首屏在手机必须先展示项目介绍再展示媒体；iPad 保持图片与介绍双栏，不能把竖图单独居中占满整行。
- 项目过程图不能直接使用整页 PPT 截图；优先提取原始局部素材，再按作品集白底、细线和暗红编号重新排版。
- 一个项目包含两个小课题时，先用分组卡片说明关系，再按小课题均衡分配过程节点。
- 小标签列表必须按内容收口，不能因为卡片等高或网格拉伸而出现大空框。
- 机械项目的重点是机械结构、传动关系和实体运行验证，不要把情景设定写成主角。
- 机械项目封面优先使用干净、有动作关系的实测图，不用临时拼接或蓝灰电机近景做首图。
- 联系方式按用户确认的公开信息展示；邮箱和手机号都要在首页档案与底部 CONTACT 可见可点。
- 用户明确范围后直接完成，不自行创建分支、扩展任务或补充虚构内容。
